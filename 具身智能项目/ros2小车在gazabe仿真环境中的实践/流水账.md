### 3. ROS2 巡检机器人任务调度与导航仿真系统（个人项目）

**2026年X月 - 2026年X月**

**背景**：
为补齐机器人中间件、导航栈和多节点系统工程能力，基于 ROS2 搭建一个轻量级巡检机器人仿真系统，覆盖节点通信、TF 坐标变换、Nav2 导航、Launch 启动编排、参数化配置、rosbag 录制与回放等 ROS2 核心工程能力。

**目标**：
构建一个可复现实验闭环：机器人在仿真环境中按照预设巡检点位依次导航，任务管理节点负责下发目标点、监听执行状态、处理失败重试与任务中断，最终形成“任务配置 → 自动导航 → 状态反馈 → 异常处理 → 数据记录”的基础机器人软件链路。

**行动**：

* 基于 ROS2 Humble、TurtleBot3 Gazebo 与 Nav2 搭建移动机器人仿真环境，完成地图加载、AMCL 定位、全局路径规划、局部避障与 RViz2 可视化配置，跑通从初始位姿设定到目标点导航的基础链路。
* 设计巡检任务配置文件，使用 YAML 管理多个目标点位、停留时间、超时时间和重试次数；实现 `patrol_task_manager` 节点，基于 `nav2_msgs/action/NavigateToPose` Action 接口依次下发目标点，并维护 `IDLE / RUNNING / SUCCEEDED / FAILED / CANCELED` 等任务状态。
* 实现任务状态发布与控制接口：通过 Topic 发布当前巡检点、导航状态、失败原因与任务进度；通过 Service 支持任务启动、暂停、恢复和取消，模拟上层业务系统对机器人任务的控制。
* 接入 `/tf`、`/odom`、`/scan` 等基础数据流，使用 RViz2、rqt_graph、tf2_tools 分析节点拓扑、坐标树和消息链路，定位导航失败、坐标系缺失和目标点不可达等常见问题。
* 使用 rosbag2 录制关键 Topic，包括 `/cmd_vel`、`/odom`、`/tf`、`/scan`、`/goal_pose` 和任务状态 Topic，用于复盘导航过程、验证任务调度逻辑，并沉淀 README、Launch 文件和参数配置，保证项目可一键复现。

**结果**：

* 完成一个轻量级 ROS2 巡检机器人仿真 Demo，支持多点巡检、状态反馈、失败重试、任务取消和 rosbag 回放。
* 在仿真环境中完成多轮巡检测试，覆盖正常导航、目标点不可达、路径受阻和任务中断等场景，验证了 ROS2 Topic / Service / Action 三类通信机制在机器人任务系统中的分工。
* 通过该项目熟悉 ROS2 节点通信、Nav2 导航栈、TF 坐标体系、Launch 编排、参数配置和基础调试工具，为后续接入真机机械臂、移动底盘或 VLA 推理服务提供机器人软件工程基础。
