## 第一部分：论文阅读与架构学习

围绕"视觉-语言-动作"统一建模路线，重点阅读并拆解以下框架：
- ACT（Action Chunking Transformer）
- π0 / π0.5 / π0.6 / π-fast

同步梳理的关键数据集：
- **LIBERO**：包含 Spatial / Object / Goal / Long，常用于语言条件下的泛化评测
- **RoboTwin**：双臂协同操作仿真数据集
- **RainbowPlus（Rainbow）**：多任务桌面操作数据集
- **RoboChallenge**

结论是：当前主流方案主要分为三类——
1）修改LLM的词表，直接预测离散的动作token
2) 以扩散或 Flow Matching 直接做动作生成；
3) 先做世界状态预测，再联合动作头优化。

第三优先：对 libero_10 单独下钻
libero_10 的 task_06/08/09 特别弱（0.3-0.4）。建议：
找出这几个具体是什么任务
06 是将马克杯放在盘子上，并处理另一个桌面物体，时间上非常紧张
08 是将两个烧水壶放在can上
09 是将黄白相见的水杯放进微波炉，还要关上，关键点在与，抓黄白水杯子时，微波炉门会卡住手臂，同时放进微波炉需要旋转手臂，这些都较难

- 核心诊断 1：训练步数与学习率调度错配。本地仅训练 **3000 steps**，而配置中 `scheduler_warmup_steps=1000`、`scheduler_decay_steps=30000`，warmup 已占总训练步数 **1/3**，训练结束时仍处于预设训练周期前段，等价于只完成了设计训练量的约 **10%**
- 核心诊断 2：图像增强配置虽已写入，但训练时未开启；官方若启用 brightness / contrast / affine 等增强，则在同任务不同随机初始场景下的泛化鲁棒性预计更强
- 核心诊断 3：弱项任务 success 序列表现出同一 task 内成功/失败交替，episode 内方差较高，说明策略对初始状态微扰敏感，训练尚不充分
- 核心诊断 4：`n_action_steps` 在 eval 时缩小，该设置未必错误，但若与官方最优评测配置不一致，会因重复推理引入更多 diffusion 采样噪声
- 改进优先级：
  - 第一优先：将训练步数由 **3000** 提升至 **50000**，并重新对齐 LR schedule
  - 第二优先：打开图像增强，优先提升任务内泛化与稳定性
  - 第三优先：针对 `libero_10` 的弱项 task 做单独下钻，必要时扩大 `chunk_size`
  - 第四优先：逐项确认 eval 参数与官方配置完全对齐

说明：当前已完成从训练到四子集分组评测的完整复现，具备稳定横向对标能力；现阶段差距已完成根因定位，后续优化方向明确，首要矛盾是训练量不足而非路线错误。
