## 第一部分：论文阅读与架构学习

围绕"视觉-语言-动作"统一建模路线，重点阅读并拆解以下框架：
- ACT（Action Chunking Transformer）
- π0 / π0.5 / π0.6 / π-fast
- InternVLA（M1 / A1）

同步梳理的关键数据集：
- **LIBERO**：包含 Spatial / Object / Goal / Long，常用于语言条件下的泛化评测
- **RoboTwin**：双臂协同操作仿真数据集
- **RainbowPlus（Rainbow）**：多任务桌面操作数据集
- **RoboChallenge**

结论是：当前主流方案主要分为三类——
1）修改LLM的词表，直接预测离散的动作token
2) 以扩散或 Flow Matching 直接做动作生成；
3) 先做世界状态预测，再联合动作头优化。

## 第三部分：架构精读（InternVLA）

### 3.1 InternVLA-A1（双臂）

A1 的核心是**三专家共享注意力**：
- `und_expert`：负责视觉语言理解（prefix，多视角图像 + 文本指令）
- `gen_expert`：负责世界状态生成（基于 Cosmos latent 预测未来帧）
- `act_expert`：负责动作生成（Flow Matching 速度场）

三路序列在每层 Transformer 中联合 QKV attention，再拆回各自分支，形成层级信息互通，不是简单串行流水线。

训练目标：
- `loss = loss_action + λ·loss_gen`
- `loss_action` 学动作速度场
- `loss_gen` 学未来帧 latent（世界模型）

### 3.2 InternVLA-M1（单臂）

M1 走的是"语义 + 空间 + 融合 + 扩散动作头"串联路线：
- Qwen2.5-VL：提供多模态语义主干
- DINOv2：补强精细空间几何感知
- 层级 QFormer：聚合压缩为固定长度动作条件
- DiT + DDIM：输出连续动作序列

对比 A1，M1 的优势是空间感知链路更清晰；A1 的优势是世界模型与动作头协同训练、更适合双臂协同扩展。

## 第五部分：阶段观察

- **M1 vs A1**：M1 偏"空间感知增强 + 扩散动作"，A1 偏"共享注意力 + 世界模型联合优化"。
- **数据集差异**：LIBERO 更适合评估语言指令泛化；RoboTwin 更贴近双臂协同与对接真机迁移需求。
- **当前瓶颈**：前沿框架迭代快、工程依赖重，复现耗时主要在环境适配与接口对齐，而非单点模型训练。
