# 毕业论文：澳洲手语轨迹分类

## 整体背景

本课题聚焦澳洲手语（Auslan）的轨迹分类与生成，目标是让系统既能识别手语动作，也能将常用词汇轨迹标准化后适配到不同体型个体上输出。

---

## 第一阶段：问题定义与开源方案复现

### 路线抉择

前期确定**以骨架关键点驱动动作识别**，而非端到端像素分类：可解释、易与后续「跨体型映射」衔接。

调研关键点提取链路：**OpenPose、HRNet、MediaPipe**（部分公开数据集已带关键点，仍把「从视频到关键点」的整条链路摸清）。

在「基于关键点的序列分类」框架下，**选定 ST-GCN 作为主复现与后续骨架建模的对照骨干**（图结构 + 时序卷积与课题路线一致）。

弄清 ST-GCN 的数据与输入形态：**官方设定下输入为 2D 关键点（x, y 与置信度）**；若使用 3D 骨架则为 (x, y, z)。常用公开设定对应数据集为 **NTU RGB+D 60** 与 **Kinetics** 系列。

### 资源约束下的复现范围

受实验室显卡条件限制（**RTX 2060 Super 8GB 显存**），无法在 NTU RGB+D 120 全量或超大 batch 上「照单全收」式复现。在 **NTU RGB+D 60** 上优先选用 **日常行为子集 Daily Actions（A1–A40，共 40 类）** 跑通 **ST-GCN** 与数据管线；该实验只作为管线验证，不作为投递简历指标。全量 60 类或更大规模留作后续有条件再补，这是**算力与周期权衡后的裁剪**，不是模型本身只能跑 40 类。

**Kinetics 系列**（如 **Kinetics-400 / 500 / 700**）在调研中已掌握：与 ST-GCN 等骨架动作论文的常见搭配、与 NTU「受控室内骨架」场景的差别等。该类数据集整体视频与样本规模约为 **NTU RGB+D 的约 7–8 倍**，存储与训练成本显著更高；**未在 Kinetics 上实际做训练复现**，仅保留文献与方案层面的认知，避免叙述成「NTU + Kinetics 双数据集均跑通」。

调研还覆盖 NTU RGB+D、NTU RGB+D 120、Kinetics 系列等在论文中的用法与数据特点；**实验复现仅落地在 NTU RGB+D 60 的上述子集**。

## 第二阶段：从简单任务起步，构建手语数字数据集

在开源侧验证后，迁移至私域 Auslan，**按难度递进**：先静态数字，再字母，再带明显肢体运动的常用词（与关键节点中的尝试顺序一致）。

### 数字（0–9）

- 使用 **MediaPipe Hand** 提取单手 21 个关键点；正面录制场景下 z 冗余，**有意只保留 2D（x, y）**，与第一阶段对 ST-GCN「2D 关键点」形态的认知一致，降低变量维度。
- 数据形态：每帧 21 个 2D 关键点，连续帧构成手势序列。
- 自采数据集：10 类，初版 475 个样本（灰度图），后经数据清洗与类别均衡扩充；按 6:2:2 随机划分训练/验证/测试集。
- 分类方法：**ST-GCN**（时序图卷积特征后接分类头）。

实验迭代过程：

| 版本 | val acc | test acc | 问题 |
|---|---|---|---|
| 初版（灰度，475样本） | 42.7% | 34.0% | 样本少 + 无归一化 |
| 彩图 + temporal=9 | 30.6% | 28.8% | 无归一化，模型崩塌预测 class 0/8 |
| +腕部归一化 + balanced | 97.5% | 97.0% | ✅ |

- 最终结果：测试集准确率 **96.97%**，10 类中有 6 类达到 100%，最差为 class 6（85.2%，主要混淆到 2/5/9）。
- 结论：骨架时序建模与预处理流程可行，归一化与类别均衡对最终指标起决定性作用，**但未回答跨说话人、双臂与伪深度问题**。

### 字母表（26 类）上的中间步骤

在拉通 50 类大关之前，曾在 **字母子集**上沿用与数字相同的 **MediaPipe Hand 2D** 流程做复现与指标摸底，结果 **92.16%**。动机是：**字母与数字同属偏静态、单手为主**，先验证「2D + 同一提取器」在私域上的天花板，再决定是否必须上 3D / 双臂 / Holistic。

## 第三阶段：Auslan 字母与常用词汇分类（50 类）与「平均 3D 关键点」失败线

### 常用词上先走的死胡同（实验失败）

在常用词汇上曾受 **平均脸（Average Face）** 思路启发，希望直接得到 **跨个体的 3D「平均手语轨迹 / 平均关键点」** 作为类原型。使用 **MediaPipe 提取的 3D 关键点** 时发现：

- **单目伪深度无法给出度量意义上的绝对深度**，跨帧、跨人叠加后误差放大；
- **不同说话人骨段长度不一**，简单对齐原点后做时间维或样本维平均，**几何意义不稳定**。

随后依次尝试过：**先验骨架长度约束、引导式动力学/正则化思路、对齐后对各关节直接求空间平均** 等变体，目标都是得到稳定的 **3D 类中心轨迹**。结论是：**在仅依赖单目 MediaPipe 3D、且不引入外部位姿与度量重建的前提下，无法可靠得到可用的 3D 平均关键点表示，该路线整体判定为失败**，写进论文与答辩时需能说明「试了什么、为何不够」。

这一失败**不否定**后续「分类」任务：分类只需判别边界，对绝对度量要求低于「生成标准轨迹」。因此**分类**与**生成**拆成两条技术债：**分类**先走归一化 + 时序模型；**生成**必须换数据源或传感（度量 3D）+ 显式骨架与 IK，即第四阶段。

### 50 类统一分类（当前主线）

在数字与字母的 2D 阶梯验证后，将 **26 字母 + 24 常用词汇** 合并为 **50 类**，统一训练，从 2D 静态为主升级为 **3D 动态序列**（常用词含双臂与上肢运动）。

- 数据集：闭源 Auslan，50 类，每类 100 个样本，共 5000 个样本，22 位说话人（各人样本数不均）。
- 关键点方案：**MediaPipe Holistic**，双手 + 上臂：每侧 21 手部 + 肩 + 肘 = 23 点，双侧共 **46 个 3D 关键点**。
- 划分方式：说话人独立切分（18 / 2 / 2）；另做按比例随机划分作对比，随机划分指标更高、存在说话人内泄漏，**以 speaker-independent 为严谨结论**。
- 模型：**LSTM** 分类器（相对 ST-GCN，在私域样本量与调参成本上更轻量，属**精度—工程成本权衡**）。
- 预处理：双肩中点对齐 + 肩宽归一化，缓解骨长差异（**这是对失败「平均 3D 点」问题的务实替代：不追求类原型，只求特征可比**）。

指标（speaker-independent 测试集）：

|              | Top-1  | Top-5  |
|--------------|--------|--------|
| 未做归一化   | 71.4%  | 89.7%  |
| 加归一化后   | 83.1%  | 96.2%  |

归一化带来约 12 个 Top-1 点，说明跨说话人对齐必要；但 **Holistic 单目伪深度**（肩 z 锚定、其余相对深度、肩带动全局漂移）在分类上可被归一化部分掩盖，**无法支撑第四阶段要的度量级轨迹与运动学指标**，因此生成侧仍必须双目（或其它度量深度方案）。
