## 项目经历

### 1. 企业级 RAG 与 Agent 问答系统（2024年6月 - 2025年6月）

**背景**：公司 400+ 员工存在财务、人力、helpdesk 等多业务线智能问答需求，系统冷启动仅 80 篇文档，且文档格式杂乱；其中个税、薪资核算等高频问题既依赖跨文档检索，也需要联动员工个体数据与规则解释工具，传统单轮 RAG 难以覆盖。

**行动**：

- 主导整体架构设计，搭建多格式文档预处理管线，使用 python-docx、python-pptx、pdfplumber、QwenVL 与 markitdown 将 Word、PPT、PDF、Excel 统一转成 Markdown+LaTeX 中间文件，并附带来源、更新时间等元数据；复杂 PPT 整页送入 QwenVL 生成结构化 Markdown，扫描件 PDF 用 QwenVL OCR，单卡 A10 吞吐约 200 篇/小时。
- 设计 Agentic RAG workflow，引入 query decomposition，将复杂问题拆解为“政策检索 + 数据查询 + 规则解释”三类子任务，按需调用公司内部接口、工资明细解释器等 function tool，形成“检索→工具调用→结果校验→答案生成→异常重试”闭环。
- 构建 BM25、Embedding、关键词多路召回与重排机制，并建立召回评测集；通过文档检索、工具执行和结果校验协同优化，1@Pass 从 45% 提升至 64%，3@Pass 从 45% 提升至 71%，ROUGE 从 0.34 提升至 0.58。
- 将个税、薪资类问题准确率从 23% 提升至 80%，并支持上传薪资条、社保截图等图片后由 QwenVL 解析字段再进入解释链路；同时将预处理管线沉淀为通用组件，支撑后续文档助手扩展至上万篇文档。

### 2. 坐席智能系统（2023年9月 - 至今）

**背景**：外呼通话场景下需同时覆盖坐席违规识别与客户意图捕捉两条链路，共 59 个语义点；初期依赖正则规则，准确率仅 60% - 70%，且每新增语义点开发周期长、维护成本高。

**行动**：

- 主导通话语义理解系统从 0 到 1 建设，先以正则规则快速上线覆盖高频场景，积累标注数据后引入 BERT 做细粒度语义点分类训练，准确率提升至 80%+，但复杂语义点仍面临高标注成本问题。
- 设计 AutoPrompt 数据构造工作流，由业务提供 100 条标注样本，使用 DeepSeek-R1 反复迭代优化 prompt，再以最优 prompt 批量标注 10000 条数据；基于标注数据对 Qwen2.5 做 LoRA 微调，并按复杂度差异化部署 7B 与 0.5B 模型，将语义点分类准确率从 60% - 70% 提升至 90%，覆盖 59 个语义点。
- 基于 Qwen2.5-14B 扩展工单填写与通话小结能力，形成“实体抽取→字段映射→结果校验”的自动化链路，工单填写准确率达到 93.5%，实现日均 9 万通通话全量覆盖，并将能力扩展至工单自动化任务。
- 沉淀词频、话术标签、情绪标签等文本特征资产，复用至后续多模态反欺诈系统文本模态，形成跨系统共享的数据与能力底座。

### 3. 数据智能与建模智能（2025年6月 - 至今）

**背景**：公司智能化转型需要在 NL2SQL、自动化建模评测（MLE-bench）与内部表格建模之间建立可复用的数据与模型链路，并形成稳定评测、训练和落地闭环。

**行动**：

- 推进 Text2SQL 基座选型与强化学习训练优化，选定 OmniSQL（Qwen2.5-Coder 7B/14B）作为基座，系统对比 DAPO 与 GRPO 在 7B/14B × LoRA/FULL 四组配置下的训练表现，主线收敛为稳定性更优的 DAPO。
- 引入重试机制、样本筛选和空结果过滤缓解奖励稀疏问题，将 7B 单次解码得分提升至 68.6，14B self-consistency 得分提升至 71.1；团队 BIRD 榜单达到 dev 71.10 / test 73.45，单模型世界第 6（2025.9）。
- 主导 MLE-bench / MLEvolve 复现，基于 MLEvolve + DeepSeek 在 MLE-bench lite 划分上复现 73 分基线；接入公司内部建模流程后，新客风控等场景 AUC 较人工建模提升约 1% - 2.5%。

### 4. AI 算力基础设施建设（2022年1月 - 至今）

**背景**：早期 AI 研发环境依赖手工配置，开发、建模、部署链路割裂，环境复现成本高；同时生产集群缺少统一 GPU 调度机制，批处理与实时推理任务并存时资源闲置与抢占问题突出。

**行动**：

- 从 0 到 1 构建支撑 50+ 人团队的统一 AI 开发平台，采用 Docker 到 K8s 的容器化架构，内置 vLLM 与多版本 PyTorch 基础镜像，支持 Jupyter 建模与 SSH 直连，将环境配置耗时由小时级缩短至 5 分钟，新员工落地周期由 4-5 天缩短至 1 天，整体研发效能提升 2 倍。
- 引入 LiteLLM 作为统一模型接入网关，结合 AIBrix 在生产集群主导 K8s GPU 调度与资源治理方案，针对大模型批处理与实时推理任务落地潮汐调度策略，实现动态资源分配，资源利用率提升 100%。

### 5. 反欺诈系统 - 多模态模块（2025年1月 - 至今）

**背景**：风控核心场景需融合通话声学、进件自拍与对话文本三路信号识别中介代办与黑产团伙，原 LightGBM 基线 AUC 仅 0.77；已有文本特征基础，但声学与图像侧缺少可规模化落地的特征链路与识别能力。

**行动**：

- 作为多模态 & NLP 算法组 Tech Leader，推动声学、人像、背景及声学年龄、图片年龄、身份证年龄差异等特征进入融合与告警链路，支撑中介代办与黑产团伙识别的差异化处置，平均每月降低资损约 60 万元。
- 主导通话音频预处理、声纹 embedding 提取及离线分析链路建设，通过流水线并行将处理吞吐量从日均 50 万通录音提升至 300 万通，支撑声学特征在风控场景中的规模化落地。
- 针对内部通话数据完成清洗、切分与声纹模型微调，将内部数据集 EER 从 19% 降至 11%；部署 Qwen-Omni 等开源多模态模型，提取静音时长、断崖式沉默等标量特征，用于反欺诈识别与客诉预测。
- 引入 QwenVL 处理耳机检测与单据金额校验等视觉审核任务，其中耳机检测经 72B 构建标注流水线、蒸馏至 7B 后微调，业务数据集准确率从 84% 提升至 90%，疑难图片准确率从 57% 提升至 78%，单据金额校验准确率达到 98%。
