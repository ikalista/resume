### 1. 企业级 RAG 与 Agent 问答系统（2024年6月 - 2025年6月）

**背景**：公司 400+ 员工存在财务、人力、helpdesk 等多业务线智能问答需求，系统冷启动仅 80 篇文档，文档格式杂乱（Word/PPT/PDF/Excel/扫描件）；其中个税、薪资核算等高频问题既依赖跨文档检索，又需要联动员工个体数据与规则解释工具，传统单轮 RAG 难以覆盖。

**行动**：

- 主导整体架构设计，搭建多格式文档预处理管线（python-docx、python-pptx、pdfplumber、QwenVL，兼用 markitdown），将 Word、PPT、PDF、Excel 统一转成 Markdown+LaTeX 标准中间文件，附带来源、更新时间等元数据 JSON；复杂 PPT 整页送入 QwenVL 生成结构化 Markdown，扫描件 PDF 用 QwenVL OCR，单卡 A10 吞吐约 200 篇/小时。
- 设计 Agentic RAG workflow，引入 query decomposition，将复杂问题拆解为“政策检索 + 数据查询 + 规则解释”三类子任务，按需调用公司内部接口、工资明细解释器等 function tool，形成“检索→工具调用→结果校验→答案生成→异常重试”闭环。
- 构建 BM25、Embedding、关键词多路召回与重排机制并建立召回评测集，支撑政策制度、流程文档和财务规则的稳定召回；通过文档检索、工具执行和结果校验协同优化，1@Pass 从 45% 提升至 64%，3@Pass 从 45% 提升至 71%，ROUGE 从 0.34 提升至 0.58。
- 将个税、薪资类问题准确率从 23% 提升至 80%，并支持上传薪资条、社保截图等图片后由 QwenVL 解析字段再进入解释链路；同时将预处理管线沉淀为通用组件，支撑后续文档助手扩展至上万篇文档。
