# 数据智能与建模智能面试稿

这个项目我会说得很直白：Text2SQL 这条线不是我原创主做的，核心思路是跟着一位 SQL 代码生成很强的同事学和做。我真正做的事情，是把她的训练思路、评测流程和工程问题完整复现一遍，把效果对齐到能解释、能复盘、能往内部场景接的程度。

这也是我进步最快的项目。因为它不是简单调一个模型，而是把 Text2SQL 的基座选择、数据生成、DAPO/GRPO 训练、推理评测、榜单指标和业务落地问题都串起来看了一遍。BIRD 上 dev 71.10、test 73.45、单模型世界第 6 是团队结果，面试里不能说成我一个人做出来的。

我亲手做和能展开讲的部分，主要是几块：第一，围绕 OmniSQL 做基座评测和训练流程复现；第二，对比 DAPO 和 GRPO 在 7B/14B、LoRA/FULL 配置下的表现；第三，处理奖励稀疏问题，包括重试机制、样本筛选和空结果过滤；第四，定位 vLLM 高 batch 推理下的 KV cache 异常，最后把 batch size 收敛到更稳定的配置；第五，用 MLEvolve + DeepSeek 在 MLE-bench lite 上复现到 73 分左右，再把这套自动建模思路接到公司内部表格建模任务里。

如果被问这个项目我最大的收获，我会说不是“我拿了一个榜单名次”，而是我第一次非常完整地复现了一个强同事的研究和工程闭环。以前我更多是自己从 0 到 1 做系统，这个项目让我意识到，跟着高手把一套方法复现到位，本身就是很快的成长方式。

被追问时的边界：

- BIRD 榜单成绩：团队成果，不说成个人独立产出。
- 个人贡献：基座评测、训练/推理流程复现、DAPO/GRPO 对比、奖励稀疏处理、vLLM 推理问题定位、MLE-bench/MLEvolve 复现和内部建模衔接。
- 可以说的亮点：我不是原作者，但我能把别人的强思路复现、吃透、工程化，并迁移到公司内部任务。
